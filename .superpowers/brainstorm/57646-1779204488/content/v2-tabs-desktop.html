How it works

Each tab = a real Terminal.app window + a claude -p session. When you create a new tab, God's Assistant spawns a Terminal window via AppleScript and positions it around the overlay. The overlay stays centered; terminals arrange in quadrants (top-left, top-right, bottom-left, bottom-right). The tab bar shows all sessions with status dots.

Spatial layout

Overlay = always centered on screen.
Terminals = positioned in quadrants around the overlay. First terminal goes top-left, second top-right, third bottom-left, fourth bottom-right. 5+ terminals stack/tile. Each terminal is a real macOS window — you can click into it, resize, interact directly.

Tab bar

Green dot = running (claude is working).
Gray dot = idle (waiting for input).
Red dot = error.
Click tab = switch focus (overlay shows that tab's activity stream, corresponding terminal window comes to front).
+ button = create new session/terminal.
× button = close session + terminal window.

Implementation plan

Phase 1: Session manager (Python) — track multiple claude -p processes, each with its own stream.
Phase 2: Tab bar (React) — switch between sessions in overlay.
Phase 3: Terminal spawner (Electron) — AppleScript to open Terminal.app, run claude commands, position windows.
Phase 4: Spatial layout — auto-position terminals around overlay in quadrants.
