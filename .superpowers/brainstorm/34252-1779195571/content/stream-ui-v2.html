Why input at the bottom

Every chat interface puts input at the bottom — iMessage, WhatsApp, ChatGPT, Slack, Discord. Users expect it there. Content flows down, you type at the bottom, new content appears above. The old layout had input near the top which breaks this mental model. Now the flow is: status strip → waveform → activity stream → response → input dock. Natural top-to-bottom reading order.

Layout anatomy (top to bottom)

1. Status strip — tiny: orb + title + status + stats. Always visible.
2. Waveform bar — thin, subtle. Flat when idle, animated when speaking.
3. Scrollable middle — Activity Stream + Response Card. This area scrolls. Auto-grows with content.
4. Input dock — pinned to bottom. Always visible. Clay pill input + mic + send + mode toggle + shortcuts.

Font: Bricolage Grotesque

Replaced Outfit with Bricolage Grotesque — a characterful, slightly quirky geometric sans with personality. Paired with Geist Mono for code/system text. Avoids the generic Inter/Outfit/Space Grotesk trap. Feels distinctive, not cookie-cutter.

Auto-resize behavior

Idle: just status strip + waveform + input dock (~140px tall). Compact.
Working: activity stream appears, window grows. Middle area scrolls if too long.
Done: activity collapses, response card shows. Window shrinks to fit.
